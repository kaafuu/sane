backend for NEC-scanners

このバックエンドは、NEC製スキャナのPC-IN500/4Cに対応しています。このバックエンドについて下記の点を覚えておいてください。作者は、使用するすべての環境下において問題なく動作するとは考えていません。また、NEC製のPC-IN500/4Cでしか動作確認ができていません（他のNEC製スキャナは動作しません）。 This backend supports the NEC scanners PC-IN500/4C. Please remember the early state of this backend; the authors do not expect it to work without problems under every condition. Especially must be pointed out that I am only able to test the backend with PC-IN500/4C, thus the current version of the backend might not work "outof the box" with another scanners.

MultiReader 1200Uは、こちらのパッチを、Microtek2のバックエンドにあてて使ってください。 MultiReader 1200U is supported by this patch for Microtek2 backend.

バージョン0.12とバージョン0.11の変更点:

  • Removed references to sharp backend (grep for "JX").
  • Check for HAVE_SYS_SHM_H before including sys/shm.h and disable shared memory support if necessary.
  • Free devlist allocated in sane_get_devices() in sane_exit() resolution setting bug fixed (PC-IN500/4C 10dpi step).
  • Removed resolution list.

バージョン0.11とバージョン0.10の変更点:

  • get_data_buffer_statusを使わないようにした。
  • tar ballをsane-1.0.3のパッチに対応。

バージョン0.10:

  • sharp-backendを元にした最初のリリース
  • 対応しているスキャナはPC-IN500/4Cだけです。
    MultiReader 300S/300S/600S/600U/650U/660U/1201U/1260Uには対応していません。
    MultiReader PetiScanは対応していません。
    MultiReader 1200Uは、こちらのパッチを、Microtek2のバックエンドにあてて使ってください。

制限事項およびわかっているバグ:

  • カラー(8bits)および、グレイ(8bits)、Lineart(1bit)のみ対応
  • ディザには未対応

今後の予定:

  • MultiReader 300S/300S/600S/600U/650U/660U/1201U/1260Uの対応。
  • MultiReader PetiScanの対応。
  • MultiReader 1200UをMicrotek2のバックエンドに取り込む。
