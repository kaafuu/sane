Unofficial patch for Microtek2 backend


I think that X12USL is only SCSI interface supported by microtek2 backend.
I write a patch for ScanMaker X12USL and NEC MultiReader 1200U for USB interface.
(MR1200U is OEM product by Microtek.)
(MR1200U is custmized for NEC and sold only in Japan.)

If you apply all my patch to the Latest microtek2 backend (or SANE CVS server) and Linux or FreeBSD.
You can use USB interface.(Support Linux and FreeBSD only)
Problems
  • Toggle lamp of flatbed is not working.
  • And, after auto lamp off, next scanning is not working.
  • Weired vertical stripes when scanning.
  • Linux driver is not use microtek X6USB driver. Because X6USB and X12USL use different USB protocol.
    X6USB uses only(?) bulk transefer. X12USL and MR1200U use control and bulk transfer.
  • I only test it on Kernel 2.2.17. (Kernel 2.5.2 is not support.)
  • I haven't find another problems.
